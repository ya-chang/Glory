## 一、前端 JS 拆分

### 现在的问题

每个 HTML 文件里都有 100~300 行内联 `<script>` 代码。改一个页面的逻辑要打开一个巨大的 HTML 文件。相同的功能（比如帖子渲染）在不同页面各写一遍。

### 改法

把内联 JS 抽到独立文件，公共逻辑提取到模块。

## 二、后端数据库加固

### 现在的问题

```js
// db.js 现在的写法
async function saveDb() {
  dirty = true;
  if (saveTimer) return;          // ← 500ms 内多次写入只执行最后一次
  saveTimer = setTimeout(async () => {
    await getClient().put(DB_FILE, Buffer.from(JSON.stringify(db)));
  }, 500);
}
```

两个用户同时发帖：
1. 用户 A 读取 db（帖子列表 = [1,2,3]）
2. 用户 B 读取 db（帖子列表 = [1,2,3]）
3. 用户 A 写入帖子 4 → db = [4,1,2,3]
4. 用户 B 写入帖子 5 → db = [5,1,2,3]  ← 帖子 4 丢了

### 改法

用文件锁 + 写入队列，不换数据库。

```js
// 改后的 db.js 核心逻辑

const writeQueue = [];
let writing = false;

// 写入队列：排队执行，不丢数据
async function enqueueSave() {
  if (writing) return;
  writing = true;
  try {
    await getClient().put(DB_FILE, Buffer.from(JSON.stringify(db, null, 2)), {
      headers: { 'Content-Type': 'application/json' }
    });
    console.log('[DB] Saved to OSS');
  } catch (err) {
    console.error('[DB] Save error:', err.message);
  } finally {
    writing = false;
    // 如果保存期间又有新的写入请求，继续处理
    if (writeQueue.length > 0) {
      writeQueue.shift();
      enqueueSave();
    }
  }
}

async function saveDb() {
  writeQueue.push(true);
  enqueueSave();
}
```

### 具体改动

| 文件 | 改动内容 |
|------|---------|
| `backend/db.js` | 把 `saveDb()` 的防抖逻辑换成写入队列 |
| `backend/db.js` | `loadDb()` 加内存缓存过期（5 分钟重新从 OSS 读取） |
| `backend/db.js` | 所有写操作（create/update/delete）改为：读→改→写 原子操作 |

### 不改的部分

- API 接口不变（`/api/forum/posts` 等路径不动）
- 数据格式不变（JSON 结构不动）
- 前端不感知这个变化

## 三、CSS 统一

### 现在的问题

```html
<!-- 这种写法到处都是 -->
<div style="display:flex;gap:12px;padding:12px 16px;background:var(--bg-subtle,#f8f9fa);border:1px solid var(--border,#e8e8e8);border-radius:6px;">
```

同一个"帖子卡片"的样式，在 `forum-home.html`、`search.html`、`profile.html` 里各写一遍，还不完全一样。

### 改法

在 `style.css` 里补一组通用 class，然后在 HTML 里替换内联 style。

### 新增 class 清单


## 七、风险和回退

```
风险：
- 抽离 JS 后页面可能有全局变量作用域问题
  → 对策：每个 page-xxx.js 用 IIFE 包裹，只暴露 init 函数

- CSS class 替换内联 style 后样式可能有细微差异
  → 对策：逐页面替换，每改一个页面就验证

- 后端写入队列可能在高并发时变慢
  → 对策：队列只是串行化写入，单次写入耗时不变

回退：
- 每改一个页面就 git commit
- 出问题可以回退到任意一步
- 后端 db.js 改动独立，可以单独回退
```
